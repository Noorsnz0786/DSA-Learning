# DSA Glossary

The canonical language for this course. Terms are added here only once the learner can
use them correctly. Definitions stay tight — one or two sentences.

## Terms

**Algorithm**:
A step-by-step procedure that takes an input and produces an output.
_Avoid_: code, logic, program

**Data structure**:
A way of organising data in memory so that specific operations on it are fast.
_Avoid_: container, storage

**Time complexity**:
How the number of steps an algorithm takes grows as the input size grows.
_Avoid_: speed, how fast it runs

**Space complexity**:
How the amount of extra memory an algorithm needs grows as the input size grows.
_Avoid_: memory usage, RAM

**Big-O**:
The notation for the *worst-case* growth rate of an algorithm, ignoring constants — e.g.
O(n) means "steps grow in direct proportion to input size n".
_Avoid_: order, complexity (use "time/space complexity" for the concept, Big-O for the notation)

**n**:
The size of the input — usually the number of items being processed.
_Avoid_: length, count (when speaking about complexity, say "n")

**Brute force**:
The most direct, usually-slowest solution that tries every possibility. The honest
starting point you state out loud before optimising.
_Avoid_: naive (acceptable, but prefer "brute force")

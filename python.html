Section 1 · The basics

Variables & Data Types

A variable is a labelled box that holds a value. Python figures out the type for you. Let's make that concrete.

What is a variable?

Real life A variable is a labelled box. You write a name on the box (age) and put something inside (25). Later you say the name and Python hands you what's inside. You can swap the contents any time. 
age = 25            # a box named "age" holding 25
name = "Noorain"    # a box holding text
age = 26            # swap the contents — totally fine
print(age)          # 26
The one rule = does NOT mean "equals" like in maths. It means "put the right-hand value into the box on the left." So x = x + 1 reads as "take what's in x, add 1, put it back" — which is why it makes sense.

The core data types

A "type" is just what kind of thing is in the box. Python's everyday types:

TypeWhat it isExample
intWhole number42, -7
floatNumber with a decimal3.14, 2.0
strText (a "string" of characters)"hello"
boolTrue or FalseTrue, False
NoneType"Nothing here yet"None
type(42)        # <class 'int'>
type(3.14)      # <class 'float'>
type("hi")      # <class 'str'>
type(True)      # <class 'bool'>
Dynamic typing — why Python feels easy Unlike some languages, you never declare the type. Python looks at the value and decides. Put a number in, it's an int; put text in, it's a str. The same box can even hold different types at different times.

Converting between types

You often need to turn one type into another — e.g. text "5" into the number 5:

int("5") + 1      # 6   (text → number)
str(5) + "!"      # "5!" (number → text)
float("3.5")      # 3.5
"5" + 1           # ERROR! can't add text and a number

Section 2 · Doing things with values

Numbers & Operators

Operators are the verbs of Python — they do things to values. Two that trip up beginners: // and %.

Arithmetic

OperatorMeansExampleResult
+ - * /add, subtract, multiply, divide7 / 23.5
//floor divide (drop the decimal)7 // 23
%modulo (the remainder)7 % 21
**power2 ** 38
Real life — // and % Sharing 7 sweets among 2 kids. Each kid gets 7 // 2 = 3 sweets (floor divide = how many whole each gets), and 7 % 2 = 1 sweet is left over (modulo = the remainder). That's exactly what these two operators give you.
Why % matters so much later % answers "is this divisible / even / odd?". n % 2 == 0 means even. i % len(arr) wraps an index back to the start (clocks, circular buffers). You'll use it constantly.

Comparison & logical operators

These give back a bool (True/False) — the basis of all decisions:

5 > 3        # True
5 == 5       # True   (== is "equals"; = is "store")
5 != 3       # True   (!= is "not equal")
(5 > 3) and (2 > 4)   # False  (and: BOTH must be true)
(5 > 3) or  (2 > 4)   # True   (or: AT LEAST ONE true)
not (5 > 3)          # False  (not: flips it)
The #1 beginner bug = stores a value; == compares. if x = 5 is an error; you want if x == 5. Say them in your head differently: "becomes" vs "is equal to".

Section 3 · Text

Strings

A string is a row of characters, each with a position (index). Slicing lets you grab any piece — let's see it visually.

Strings are indexed rows of characters

Real life Think of a string as a row of lockers, each holding one character, numbered from 0. Python also lets you count from the end with negative numbers: -1 is the last character.

🎚️ Animation — slicing a string

Click a slice and watch which characters it grabs. s[start:stop] takes from start up to but not including stop.

s = "PYTHON"
Click a slice above.
The slice rule (memorise) s[a:b] = from index a up to but not including b. Leave out a → start from the beginning. Leave out b → go to the end. s[::-1] reverses the whole string.

Useful string tools

s = "Hello World"
len(s)            # 11  (how many characters)
s.lower()         # "hello world"
s.upper()         # "HELLO WORLD"
s.split()         # ["Hello", "World"]  (break into a list)
"-".join(["a","b","c"])  # "a-b-c"  (glue a list into text)
"World" in s      # True  (is it inside?)
s.replace("World","Python")  # "Hello Python"

f-strings — the clean way to build text

name = "Noorain"; age = 26
f"My name is {name} and I am {age}"   # "My name is Noorain and I am 26"
Strings are immutable You cannot change a character in place: s[0] = "h" is an ERROR. Methods like .lower() return a new string; they never edit the original. To "change" a string, you reassign: s = s.lower().

Section 4 · The workhorse

Lists

A list is an ordered, changeable collection — the single most-used structure in Python. Master it and most code clicks.

What is a list?

Real life A list is a numbered shopping list you can edit: add items, cross some off, reorder them. Each item has a position (index from 0), and you can change the contents any time. 
nums = [10, 20, 30]
nums[0]            # 10   (first item)
nums[-1]           # 30   (last item)
nums[1:3]          # [20, 30]  (slicing works just like strings)

Changing a list (it's mutable)

nums = [10, 20, 30]
nums.append(40)    # [10, 20, 30, 40]  add to the end
nums.pop()         # removes & returns 40 → [10, 20, 30]
nums.insert(1, 99) # [10, 99, 20, 30]  insert at index 1
nums.remove(99)    # [10, 20, 30]  remove first matching value
len(nums)          # 3
30 in nums         # True
List vs string — the key difference Strings are immutable (can't change in place). Lists are mutable — nums[0] = 99 works fine. That's why lists are the go-to when you need to build up or modify a collection.

Sorting & handy tools

nums = [3, 1, 2]
sorted(nums)       # [1, 2, 3]  returns a NEW sorted list
nums.sort()        # sorts nums itself, in place
sum(nums)          # 6
min(nums), max(nums)  # (1, 3)
sorted() vs .sort() sorted(nums) returns a brand-new sorted list and leaves the original alone. nums.sort() changes nums in place and returns None. Mixing these up (x = nums.sort() gives you None!) is a classic beginner trap.

Section 5 · Key → value

Dictionaries

A dictionary stores pairs: a key and its value. Look up by key and get the value instantly. (This is the same thing DSA calls a hash map!)

What is a dictionary?

Real life A real dictionary (or a phone book): you look up a word (the key) and instantly get its meaning (the value) — without reading every page. Python's dict works exactly like that. 
person = {"name": "Noorain", "age": 26}
person["name"]          # "Noorain"  (look up by key)
person["age"] = 27      # change a value
person["city"] = "Pune" # add a new pair
"name" in person        # True  (is this key present?)
person.get("phone", "n/a")  # "n/a"  (safe lookup with a default)
Why dictionaries are everywhere Looking something up by key is instant (no scanning). That makes dicts perfect for "remember what I've seen", counting things, and mapping one thing to another. In DSA this exact tool (the hash map) turns slow O(n²) solutions into fast O(n) ones — see the DSA tab.

Counting with a dictionary (super common)

text = "banana"
counts = {}
for ch in text:
    counts[ch] = counts.get(ch, 0) + 1   # add 1, defaulting to 0
print(counts)    # {'b': 1, 'a': 3, 'n': 2}

Looping over a dictionary

for key in person:              # keys
    print(key)
for key, value in person.items():   # key AND value together
    print(key, value)

Section 6 · Repeating work

Loops

A loop repeats a block of code. This is where beginners get confused about "how it flows" — so let's watch it run, one step at a time.

The for loop — do something for each item

Real life A teacher taking attendance: for each student in the room, call their name. The loop does the same action once per item, with the loop variable holding "the current one". 
for fruit in ["apple", "banana", "cherry"]:
    print(fruit)
# apple
# banana
# cherry

🎚️ Animation — watch a loop add up a list

Step through total = 0; for x in nums: total += x. Watch the current item light up and the running total grow.

nums = [10, 20, 30, 40]
x = –  ·  total = 0

range() — loop a fixed number of times

for i in range(3):     # 0, 1, 2  (stops BEFORE 3)
    print(i)
for i in range(1, 4):  # 1, 2, 3
    print(i)

enumerate — when you need the index too

for i, fruit in enumerate(["apple", "banana"]):
    print(i, fruit)
# 0 apple
# 1 banana

The while loop & break / continue

n = 5
while n > 0:        # keep going WHILE the condition is true
    print(n)
    n -= 1          # ⚠ must move toward stopping, or it loops forever!

for x in nums:
    if x == 30: break      # stop the loop entirely
    if x == 20: continue   # skip to the next item
The infinite-loop trap A while loop runs as long as its condition is true. If nothing inside ever makes it false (you forgot n -= 1), it runs forever. Always ask: "what makes this loop stop?"

Section 7 · Making decisions

Conditionals (if / elif / else)

Code that chooses a path based on a condition. This is how programs make decisions.

The idea

Real life A bouncer at a club: if you are over 18, let you in; else turn you away. The condition decides which path the code takes. 
score = 75
if score >= 90:
    print("A")
elif score >= 70:    # "else if" — checked only if the first was false
    print("B")
else:                # runs if NONE of the above were true
    print("C")
# B
Indentation IS the structure Python uses indentation (the spaces) to know what is inside the if. There are no curly braces. Line up your blocks consistently (4 spaces) or you get an error. Exactly one branch of an if/elif/else runs.

Truthiness — what counts as "true"

Falsy values (treated as False) 0, 0.0, "" (empty string), [] (empty list), {} (empty dict), and None are all falsy. Everything else is truthy. So if my_list: means "if the list is not empty". 
age = 20
status = "adult" if age >= 18 else "minor"   # one-line if/else (ternary)

Section 8 · Fixed collections

Tuples

A tuple is like a list that can never change — perfect for fixed records and returning several values at once.

The idea

Real life A tuple is a sealed envelope: coordinates (3, 7) or a date (26, 6, 2026) — a fixed group of values that belong together and should not be edited. 
point = (3, 7)        # a tuple, made with parentheses
point[0]              # 3   (index like a list)
len(point)            # 2
point[0] = 9          # ERROR — tuples are immutable
single = (5,)         # a one-item tuple NEEDS the comma

Unpacking — the real superpower

x, y = (3, 7)         # x = 3, y = 7  (unpacking)
a, b = 1, 2
a, b = b, a           # swap in one clean line!
Where you will meet tuples Functions return several values as a tuple (return x, y). Looping for k, v in d.items() unpacks tuples. And because they are immutable, tuples can be used as dictionary keys (lists cannot).

Section 9 · Unique collections

Sets

A set is a bag of unique items with no order — perfect for removing duplicates and lightning-fast "have I seen this?" checks.

The idea

Real life A guest list where each name can appear only once. Add a duplicate and nothing happens — it is already there. There is no "first" or "second" guest; order does not matter. 
seen = set()          # empty set (NOT {} — that is a dict!)
seen.add(3)
seen.add(3)           # ignored — already present
print(seen)           # {3}
3 in seen             # True — membership check is O(1), super fast
set([1, 1, 2, 3])     # {1, 2, 3}  — duplicates removed instantly
The two killer uses (1) Remove duplicates from a list: set(my_list). (2) Fast membership: checking x in a_set is O(1), versus O(n) for a list. This is the same idea behind hash maps in the DSA tab. 
{1, 2} | {2, 3}   # {1, 2, 3}  union (everything)
{1, 2} & {2, 3}   # {2}        intersection (common)
{1, 2, 3} - {2}   # {1, 3}     difference

Section 10 · Reusable blocks

Functions

A function is a named recipe: give it inputs, it does steps, and hands back a result. Write once, use anywhere.

The idea

Real life A coffee machine: you put in beans and water (the arguments), it runs its steps, and gives back coffee (the return value). You do not rebuild the machine each time — you just use it. 
def add(a, b):        # def names it; a, b are PARAMETERS (inputs)
    result = a + b
    return result     # hand the answer back to whoever called it

x = add(3, 4)         # CALL it with ARGUMENTS 3 and 4
print(x)              # 7
return vs print — the biggest beginner confusion print just shows something on screen. return hands a value back so the rest of your code can use it. A function that prints but does not return gives back None — so x = add(3,4) would be None if add only printed.

Default values & multiple returns

def greet(name, greeting="Hi"):   # greeting has a DEFAULT
    return f"{greeting}, {name}"
greet("Sam")           # "Hi, Sam"
greet("Sam", "Hello")  # "Hello, Sam"

def stats(nums):
    return min(nums), max(nums)   # returns a tuple
low, high = stats([4, 1, 9])      # unpack it
